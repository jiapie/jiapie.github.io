# 03.機器學習技術與應用

## 1. 機器學習基礎數學

### 機率／統計基礎

**機率在機器學習中的核心邏輯：**

機器學習模型的本質是學習條件機率分佈 **P(Y|X)**，即在已知輸入特徵 X 的情況下，預測目標 Y 的可能性分佈。這種機率視角讓模型能夠：
- 輸出預測信心度（而非只給確定答案）
- 評估不確定性，支持風險控制與決策制定

**常見機率分佈在 ML 的應用：**

| 分佈 | ML 應用場景 |
|------|-----------|
| **伯努利分佈** | 邏輯迴歸的目標變數（二元分類）|
| **多項式分佈** | 多類別分類（Softmax 輸出）|
| **常態分佈** | 線性迴歸假設誤差項服從常態；特徵標準化基礎；生成模型的先驗 |
| **均勻分佈** | 神經網路的隨機初始化參數 |
| **VAE 潛在空間** | 變分自動編碼器將潛在變數建模為常態分佈 N(μ,σ²) |

**條件機率（Conditional Probability）：**

$$P(A|B) = \frac{P(A \cap B)}{P(B)}$$

在分類任務中，模型學習的核心就是 P(Y|X)：在輸入 X 條件下，各類別 Y 的發生機率。

**貝氏定理（Bayes' Theorem）：**

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

| 元素 | 名稱 | 說明 |
|------|------|------|
| **P(A)** | 先驗機率（Prior） | 觀察 B 之前對 A 的初始信念 |
| **P(B\|A)** | 似然（Likelihood） | A 發生時，B 出現的可能性 |
| **P(B)** | 邊際機率（Marginal） | B 發生的整體機率（正規化因子）|
| **P(A\|B)** | 後驗機率（Posterior） | 觀察 B 之後，更新後的 A 的機率 |

**貝氏在 ML 的應用：**
- **樸素貝氏分類器**：計算 P(類別|特徵)，用於垃圾郵件過濾、文本分類
- **貝氏優化（Bayesian Optimization）**：用先驗模型指導超參數搜尋
- **生成模型**：VAE、隱馬可夫模型（HMM）中建構變數間的依存關係

**統計量在 ML 特徵工程的應用：**

| 統計量 | ML 應用 |
|-------|---------|
| **期望值** | 線性迴歸截距估計；特徵均值填補缺失值 |
| **變異數/標準差** | 判斷特徵是否需要標準化（避免尺度偏誤）|
| **偏態** | 偏態過大 → 對數轉換或 Box-Cox 轉換改善收斂 |
| **峰度** | 高峰度（重尾）→ 異常值偵測與風險評估的信號 |
| **相關係數** | 識別冗餘特徵，進行特徵篩選 |

### 線性代數基礎

**向量（Vector）在 ML 的角色：**
- 一筆樣本的特徵表示：**x** = [x₁, x₂, ..., xₙ]ᵀ
- 模型參數向量：**θ** = [θ₁, θ₂, ..., θd]，預測值 ŷ = **θ**ᵀ**x**
- 重要運算：點積（Dot Product，計算相似度）、L2 範數（向量長度，用於正則化）

**矩陣（Matrix）在 ML 的角色：**

| 應用 | 形式 | 說明 |
|------|------|------|
| **特徵矩陣** | X ∈ ℝⁿˣᵈ | n 筆樣本 × d 個特徵，整批資料的數學表示 |
| **權重矩陣** | W ∈ ℝᵈˣᵏ | 神經網路層間的連接權重，d 維輸入 → k 維輸出 |
| **線性迴歸預測** | ŷ = Xθ | 矩陣乘法一次計算所有樣本的預測值 |
| **神經網路前向傳播** | z⁽ˡ⁾ = W⁽ˡ⁾a⁽ˡ⁻¹⁾ + b⁽ˡ⁾ | 每層運算本質是矩陣乘法加上偏差向量 |

**線性變換（Linear Transformation）：**

矩陣左乘向量 Ax 代表對原始向量進行線性變換，幾何意義包括：
- **縮放**：調整向量大小
- **旋轉**：改變向量方向
- **投影**：將高維向量投射到低維子空間（PCA 的核心操作）

**矩陣分解與降維：**

| 方法 | 原理 | 應用 |
|------|------|------|
| **特徵值分解（EVD）** | 對稱方陣 A = QΛQᵀ；找「拉伸方向」與重要性 | PCA、LDA |
| **奇異值分解（SVD）** | 任意矩陣 X = UΣVᵀ；更通用的分解 | PCA 計算、推薦系統、潛在語意分析（LSA）、影像壓縮 |
| **非負矩陣分解（NMF）** | X ≈ WH（所有元素非負）；部件疊加的解釋方式 | 主題建模、生物訊號分析 |

**降維的 ML 好處：**
- 提升訓練效率（減少參數量與計算資源）
- 降低過擬合風險（去除雜訊與共線性）
- 強化可解釋性（轉換後的特徵有幾何語意）
- 支持資料視覺化（投影到 2D/3D）

**最小平方估計（OLS，Ordinary Least Squares）：**
- 線性迴歸的參數求解方法：找一組 θ 使預測誤差平方和最小
- 幾何意義：將輸入資料投影到最接近實際結果的超平面

### 數值優化技術與方法

**優化問題的三要素：**
1. **目標函數（損失函數）**：衡量預測偏差，如 MSE、Cross-Entropy
2. **決策變數**：模型可調整的參數（線性回歸的 θ、神經網路的 W、b）
3. **可行域**：參數允許的範圍（如非負條件、L1 稀疏約束）

**損失函數對應任務：**

| 任務 | 損失函數 | 特性 |
|------|---------|------|
| **迴歸** | MSE（均方誤差）| 放大大誤差懲罰；受離群值影響大 |
| **迴歸** | MAE（平均絕對誤差）| 對離群值穩健；收斂較慢 |
| **迴歸** | Huber Loss | MSE+MAE 的折衷；小誤差用 MSE，大誤差用 MAE |
| **二元分類** | Binary Cross-Entropy | 衡量預測機率與真實標籤的距離 |
| **多分類** | Categorical Cross-Entropy | Softmax 輸出後的標準損失 |
| **不平衡分類** | Focal Loss | 加重對難分類樣本的懲罰 |

**梯度下降（Gradient Descent）三種變體：**

| 方法 | 每次用多少資料更新 | 特點 | 適用情境 |
|------|----------------|------|---------|
| **批次 GD（Full Batch）** | 全部訓練資料 | 更新穩定但速度慢、記憶體消耗大 | 小型資料集 |
| **隨機 GD（SGD）** | 單一樣本 | 更新快但波動大、難收斂 | 大型資料集、線上學習 |
| **小批次 GD（Mini-batch SGD）** | 小批資料（32/64/128）| 速度與穩定性平衡，主流選擇 | 深度學習標準訓練方式 |

**進階優化器比較：**

| 優化器 | 核心機制 | 優點 | 適用場景 |
|-------|---------|------|---------|
| **Momentum** | 累積歷史梯度方向（物理動量類比）| 克服震盪，加速收斂 | 深層神經網路 |
| **Adagrad** | 依歷史梯度大小自動調整學習率 | 適合稀疏特徵 | NLP 稀疏輸入 |
| **RMSprop** | 滑動平均修正 Adagrad 的學習率過快下降 | 穩定非凸訓練 | RNN、非平穩梯度 |
| **Adam** | 結合 Momentum + RMSprop；追蹤一、二階動量 | 訓練快、收斂穩；目前最常用 | 幾乎所有深度學習任務 |

**收斂判準（何時停止訓練）：**
- 損失函數變化趨緩（連續多次迭代變化量 < ε）
- 驗證集效能不再提升
- 梯度值趨近於零
- 達到設定的最大訓練輪數（epoch 上限）

**訓練穩定性策略：**

| 策略 | 說明 |
|------|------|
| **學習率排程（LR Scheduling）** | 訓練後期自動降低學習率（Step Decay / Cosine Annealing）|
| **Early Stopping** | 驗證集效能連續 N 個 epoch 不改善則停止，防過擬合 |
| **梯度裁剪（Gradient Clipping）** | 限制梯度最大值，防止 RNN/深層網路梯度爆炸 |
| **Batch Normalization** | 標準化每層輸入分佈，加速收斂、提升穩定性 |

**凸性（Convexity）對訓練的影響：**
- **凸函數**（如線性迴歸、邏輯迴歸的損失）：任何局部最小值即為全域最小值，訓練有保證
- **非凸函數**（如神經網路損失）：存在多個局部最小值與鞍點，需靠隨機初始化、動量機制等應對

## 2. 機器學習與深度學習

### 機器學習原理與技術

**三大學習典範：**

| 類型 | 輸入資料 | 目標 | 典型適用場景 |
|------|---------|------|------------|
| **監督學習** | 有標籤 (x, y) | 學習映射 f(x)→y | 垃圾郵件分類、房價預測、醫療診斷 |
| **非監督學習** | 無標籤 (x) | 發現隱藏結構 | 客群分群、異常偵測、降維、主題模型 |
| **強化學習** | 環境回饋（獎勵/懲罰） | 最大化累積獎勵 | 遊戲 AI（AlphaGo）、機器人控制、自動駕駛 |

**監督學習細分：**
- **回歸（Regression）**：輸出連續值（房價、股價預測）
- **分類（Classification）**：輸出離散類別（是/否、多類別）

**非監督學習細分：**
- **分群（Clustering）**：K-Means、DBSCAN
- **降維（Dimensionality Reduction）**：PCA、t-SNE
- **密度估計**：學習資料分佈

**半監督學習（Semi-supervised Learning）：**
- **特性**：少量標注資料 + 大量未標注資料共同訓練
- **適用**：標注成本高但未標注資料豐富（如醫療影像、農業病害辨識）
- **常見方法**：偽標籤法（Pseudo-Labeling）、自訓練（Self-Training）、一致性正則化

**自監督學習（Self-supervised Learning）：**
- **特性**：從資料本身**自動產生**訓練目標，不需人工標注
- **常見任務**：遮罩語言建模（BERT 的 MLM）、預測下一個 Token（GPT）、對比學習（SimCLR）
- **重要性**：大型語言模型（LLM）預訓練的核心機制
- **與監督學習的差異**：自監督學習**不需人工逐筆提供標注資料**，標籤由資料結構自動生成

**強化學習關鍵概念：**
- **Agent（智能體）**、**Environment（環境）**、**State（狀態）**、**Action（動作）**、**Reward（獎勵）**
- **策略（Policy）**：決定 Agent 在某狀態下採取哪個動作
- **價值函數（Value Function）**：估計從某狀態出發的長期累積獎勵

**核心演算法：**

| 類別 | 演算法 | 說明 |
|------|--------|------|
| **值基礎（Value-Based）** | Q-Learning | 學習每個狀態-動作對的 Q 值（期望報酬），選 Q 值最大的動作 |
| | **DQN**（Deep Q-Network） | Q-Learning + 深度神經網路，DeepMind 用來攻克 Atari 遊戲 |
| **策略基礎（Policy-Based）** | REINFORCE | 直接最佳化策略函數，適合連續動作空間 |
| | **PPO**（Proximal Policy Optimization） | 目前最穩定的策略梯度演算法，ChatGPT RLHF 所使用 |
| **演員-評論家（Actor-Critic）** | A3C / SAC | 結合值基礎與策略基礎的優點 |

**Q-Learning 更新公式：**
```
Q(s, a) ← Q(s, a) + α × [r + γ × max Q(s', a') - Q(s, a)]
```
- α：學習率；γ：折扣因子（重視長期 vs 短期回報）；r：即時獎勵

**實際應用案例：**

| 案例 | 技術 | 成就 |
|------|------|------|
| **AlphaGo / AlphaZero** | Monte Carlo Tree Search + DRL | 擊敗人類圍棋世界冠軍 |
| **OpenAI Five** | PPO | 擊敗職業 Dota 2 選手 |
| **ChatGPT RLHF** | PPO + 人類回饋 | 對齊語言模型輸出與人類偏好 |
| **自動駕駛** | DRL + 模擬環境 | 車道保持、交叉路口決策 |
| **機器人控制** | SAC / PPO | 機械手臂抓取、步行機器人 |
| **推薦系統** | DRL | 動態調整推薦序列以最大化點擊／留存率 |

**探索與利用（Exploration vs Exploitation）：**
- **Exploitation**：選目前已知最好的動作（貪婪策略）
- **Exploration**：嘗試未知動作以發現更好的策略
- **ε-Greedy 策略**：以 ε 機率隨機探索，1-ε 機率選最佳動作；訓練過程逐步降低 ε

### 常見機器學習演算法

**線性回歸（Linear Regression）：**
- 假設輸出與特徵為線性關係：`y = Wx + b`
- 損失：MSE（均方誤差）；優化：梯度下降

**邏輯回歸（Logistic Regression）：**
- 線性模型 + Sigmoid 函數 → 二元分類機率
- 易解釋、訓練快，是分類任務的基線模型

**支持向量機（SVM）：**
- 找到最大化邊界（Margin）的超平面
- **核函數（Kernel）**：RBF、多項式核 → 處理非線性問題
- 適合：高維小樣本、文字分類；不適合：超大資料集

**決策樹（Decision Tree）：**

決策樹以樹狀結構遞迴分割特徵空間，每個內部節點代表一個特徵的判斷條件，葉節點代表最終類別或數值預測。

| 演算法 | 分割準則 | 特性 |
|--------|---------|------|
| **ID3** | 資訊增益（Information Gain） | 偏好多分支屬性，不支援連續值 |
| **C4.5** | 增益率（Gain Ratio） | 修正 ID3 偏差，支援連續值與缺失值 |
| **CART** | Gini 不純度 | 二元分割，支援回歸與分類，sklearn 預設 |

**關鍵概念：**
- **資訊增益**：分割前後 Entropy 的減少量，越大越好
- **Gini 不純度**：`1 - Σp²`，越小越純
- **過擬合防治**：
  - 預剪枝（Pre-pruning）：限制最大深度、最小樣本數
  - 後剪枝（Post-pruning）：建完整樹後裁除貢獻低的子樹

**集成方法（Ensemble）：**
- **隨機森林（Random Forest）**：多棵決策樹 + Bagging，降低方差
- **XGBoost / LightGBM**：Gradient Boosting，逐步修正錯誤，目前最強的表格資料模型

**K 近鄰（K-NN）：**
- 根據距離最近的 K 個鄰居投票決定類別
- 惰性學習（Lazy Learning）：無訓練過程
- 缺點：預測慢（O(n)）、高維度效果差（維度詛咒）
- K 值選擇：K 小 → 容易過擬合；K 大 → 容易欠擬合

**主成分分析（PCA）：**
- 找到方差最大的方向（主成分），將資料投影到低維空間
- 目的：降維、去除雜訊、視覺化
- 步驟：標準化 → 計算協方差矩陣 → 特徵分解 → 選取前 k 個主成分

### 深度學習原理與框架

**類神經網路（ANN）結構：**

```
輸入層（Input Layer）→ 隱藏層 × N（Hidden Layers）→ 輸出層（Output Layer）
```

每個神經元計算：`output = activation(W · x + b)`

**學習機制（監督式）：**
1. **前向傳播**：資料逐層計算，得到預測值
2. **損失計算**：比較預測值與真實標籤（Cross-Entropy、MSE）
3. **反向傳播**：鏈式法則逐層計算梯度
4. **權重更新**：`W ← W - η × ∂L/∂W`（梯度下降）
5. 重複直到損失收斂

**防止過擬合技術：**

| 技術 | 說明 |
|------|------|
| **Dropout** | 訓練時隨機關閉部分神經元，防止共適應 |
| **L1/L2 正規化** | 對權重加入懲罰項，抑制過大權重 |
| **Batch Normalization** | 正規化每層輸入，加速收斂 |
| **Early Stopping** | 監控驗證損失，於最低點停止訓練 |

（CNN、RNN、GAN 等進階架構詳見科目一：AI 技術應用與規劃）

**知識蒸餾（Knowledge Distillation）：**
- **原理**：讓小型「學生模型（Student）」模仿大型「教師模型（Teacher）」的輸出分布，學習到近似效果但參數量更少
- **目的**：在維持模型效能的前提下，**降低推論成本與延遲**
- **應用**：在資源受限的邊緣設備部署、RAG 系統效能優化（以小模型降低推論成本）
- **與 PEFT 差異**：PEFT（LoRA等）是訓練時只更新少量參數；蒸餾是訓練一個全新的小模型

**Wide & Deep 模型：**
- **架構**：同時包含「Wide（寬）」線性部分 +「Deep（深）」神經網路部分
- **Wide 部分**：學習已出現的特徵組合規律（記憶，Memorization）
- **Deep 部分**：對未出現的特徵組合做泛化預測（泛化，Generalization）
- **適用**：同時包含大量稀疏類別特徵與數值型特徵的資料（如電商推薦、廣告點擊率預測）

## 3. 機器學習建模與參數調校

### 數據準備與特徵工程

**機器學習訓練完整流程：**

```
資料收集 → 資料前處理 → 特徵工程 → 模型選擇 → 訓練 → 評估 → 超參數調整 → 部署
```

**資料前處理（Data Preprocessing）：**

| 步驟 | 方法 |
|------|------|
| 缺失值處理 | 刪除、均值/中位數填補、KNN 插補 |
| 異常值處理 | Z-score、IQR 法偵測並處理 |
| 特徵縮放 | 標準化（StandardScaler）、Min-Max 正規化 |
| 類別編碼 | One-Hot Encoding、Label Encoding |
| 資料分割 | Train / Validation / Test（常用 70/15/15 或 80/10/10） |

**特徵工程：**
- 特徵選擇：移除低方差、高相關特徵（Filter/Wrapper/Embedded 法）
- 特徵構造：多項式特徵、交叉特徵
- 降維：PCA、LDA

**處理偏態特徵（Skewed Features）：**
- 金額、收入等特徵常呈右偏態，少數極大值影響模型穩定性
- **對數轉換（Log Transformation）**：`x' = log(x + 1)`，壓縮極端值、改善常態性
- 其他方法：Box-Cox 轉換、平方根轉換

**處理類別不平衡（Class Imbalance）：**

| 方法 | 說明 |
|------|------|
| **SMOTE** | 在少數類別的特徵空間**合成新樣本**（插值方式），增加少數類別樣本量 |
| **欠採樣** | 隨機刪除多數類別樣本，降低類別失衡程度 |
| **調整類別權重** | 在損失函數中提高少數類別的懲罰權重 |
| **Focal Loss** | 自動加重難分類樣本的損失，減少易分類樣本的影響 |

> **考試陷阱：** SMOTE 是「合成新樣本」，不是刪除多數類別、也不是調整損失函數權重。

**防止資料洩漏（Data Leakage）：**
> 資料洩漏：在模型訓練/評估時，測試集資訊「洩漏」進入訓練過程，導致評估結果虛高。

**正確做法：先分割資料，再用訓練集計算統計量，然後套用到測試集**

```
❌ 錯誤：計算全資料集的均值/標準差 → 再分割 → 標準化（測試資訊洩漏進訓練）
✅ 正確：先分割 Train/Test → 僅用訓練集計算均值/標準差 → 套用到測試集
```

常見資料洩漏場景：
- 標準化前未先分割資料
- 特徵選擇時使用了全資料集
- 目標變數的未來資訊洩漏到特徵中（時間序列）

### 模型選擇與架構設計

**如何選擇模型：**

| 問題類型 | 資料規模 | 推薦模型 |
|---------|---------|---------|
| 二元分類 | 小 | Logistic Regression、SVM |
| 多分類 | 中 | 決策樹、隨機森林 |
| 回歸 | 中大 | XGBoost、神經網路 |
| 影像 | 大 | CNN（ResNet、EfficientNet） |
| 序列/文字 | 大 | RNN/LSTM、Transformer |

**偏差-方差權衡（Bias-Variance Tradeoff）：**
- **高偏差（欠擬合）**：模型過於簡單 → 增加模型複雜度、增加特徵
- **高方差（過擬合）**：模型過於複雜 → 正規化（L1/L2）、Dropout、增加資料

### 模型訓練、評估與驗證

**評估指標：**

| 指標 | 用途 |
|------|------|
| Accuracy | 一般分類（類別平衡時） |
| Precision / Recall | 類別不平衡（如詐欺偵測） |
| F1-Score | Precision 與 Recall 的調和平均 |
| AUC-ROC | 分類器整體效能 |
| MSE / RMSE / MAE | 回歸任務 |

**驗證方法：**
- **Hold-out**：簡單切分，速度快
- **K-Fold 交叉驗證**：更穩健的泛化能力評估
- **Stratified K-Fold**：適合類別不平衡資料

---

### 模型調整與優化

**超參數調整方法：**

| 方法 | 說明 |
|------|------|
| **Grid Search** | 窮舉所有組合；適合小搜尋空間 |
| **Random Search** | 隨機取樣；比 Grid Search 更高效 |
| **Bayesian Optimization** | 利用先前結果推斷最優方向，效率最高 |

**常見超參數：**
- 學習率（Learning Rate）：最關鍵，過大震盪、過小收斂慢
- 批次大小（Batch Size）：影響訓練穩定性與速度
- 訓練輪次（Epochs）+ 早停（Early Stopping）
- 正規化係數（λ）：控制 L1/L2 懲罰強度
- 樹的深度、K-NN 的 K 值

**優化器（Optimizer）：**
- **SGD**：隨機梯度下降，基礎
- **Adam**：自適應學習率，目前最常用
- **RMSProp**：適合 RNN
