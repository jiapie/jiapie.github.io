## 1. 人工智慧概念

### AI 的定義與分類

**定義：** 人工智慧（AI）是模擬人類智慧行為的技術，使機器能夠執行學習、推理、解決問題、感知環境等原本需要人類智慧的任務。

**依功能分類：**

| 類型 | 說明 | 應用 |
|------|------|------|
| **分析型 AI** | 洞悉數據模式，分析大量資料提供見解 | 商業智慧、市場分析 |
| **預測型 AI** | 基於歷史資料預測未來趨勢 | 市場預測、風險評估 |
| **生成型 AI** | 根據提示詞（Prompt）生成文字、語音、圖像、影片 | ChatGPT、Midjourney |

**依智能程度分類：**

| 類型 | 定義 | 現況 |
|------|------|------|
| **弱人工智慧（Narrow AI）** | 專注於單一任務，如語音助理、影像辨識 | 目前產業應用主流 |
| **強人工智慧（General AI）** | 具備與人類相當的推理與學習能力，可處理跨領域問題 | 尚未實現 |
| **超人工智慧（Super AI）** | 超越人類智慧，具備自主學習與創造力 | 僅存在理論與科幻構想 |

**AI 技術底層架構：**
```
資料處理與分析（ETL、Hadoop、Spark）
        ↓
演算法（回歸、分類、決策樹）
        ↓
機器學習（監督 / 非監督 / 強化）
        ↓
深度學習（DNN、CNN、RNN）
        ↓
應用開發（模型設計、訓練、部署）
        ↓
實際運用（智慧醫療、智慧製造、金融風控）
```

**各行業 AI 應用：**
- **醫療**：影像診斷、藥物研發（AlphaFold）、個人化治療
- **金融**：信用評估、詐欺偵測、自動交易
- **製造**：自動化生產、品質檢測、預測性維護
- **交通**：自動駕駛、交通流量預測
- **娛樂**：遊戲 AI、內容推薦、VR 沉浸體驗

---

### AI 治理概念

**負責任的 AI（Responsible AI）六大核心原則：**

| 原則 | 說明 |
|------|------|
| **公平性與無偏見** | 避免因訓練資料偏差導致歧視或不公平決策 |
| **透明性與可解釋性** | AI 決策過程可被追溯，避免「黑箱問題」（XAI 技術） |
| **安全性與穩健性** | 具備防錯能力，對抗攻擊輸入仍能穩定運作 |
| **責任與治理** | 明確 AI 出錯時的責任歸屬，建立監管稽核機制 |
| **隱私與安全** | 遵循 GDPR、CCPA、台灣個資法，保護個人資料 |
| **以人為中心** | AI 輔助人類決策，非完全取代人類判斷 |

## 2. 資料處理與分析概念

### 資料基本概念與類型

| 類型 | 說明 | 格式 | 範例 |
|------|------|------|------|
| **結構化資料** | 固定行列格式，易於查詢分析 | SQL、Excel | 交易記錄、ERP 資料 |
| **半結構化資料** | 有標籤但格式靈活 | JSON、XML、CSV | IoT 設備紀錄、API 回傳 |
| **非結構化資料** | 無固定結構，需 AI 技術處理 | 圖片、音訊、文字 | 醫療影像、社群輿情 |

> 資料品質決定 AI 效能上限：「垃圾進，垃圾出（Garbage In, Garbage Out）」

### 資料整理與分析流程（ETL）

```
資料收集 → 資料清理 → 資料轉換 → 資料統一 → 探索性分析（EDA）→ 建模
```

**資料收集來源：** 問卷調查、自有產品數據、開放資料 API、網路爬蟲、第三方付費資料

**資料清洗四大問題：**

| 問題 | 解法 |
|------|------|
| **遺缺值（Missing Value）** | 均值/中位數填補、插補法、刪除記錄 |
| **重複值（Duplicate Value）** | 識別主鍵，保留一筆正確記錄 |
| **錯誤值（Error Value）** | 超出合理範圍（如年齡 -5）→ 修正或刪除 |
| **離群值（Outlier Value）** | Z-score、IQR 偵測；依業務需求決定保留或移除 |

**資料轉換技術：**
- 格式轉換（CSV → JSON）
- 正規化/標準化（縮放至 0–1 或 Z-score）
- 離散化（年齡分組：青年/中年/老年）
- 降維（PCA 減少特徵維度）

**探索性資料分析（EDA）：**
- 描述性統計：平均值、中位數、標準差、百分位數
- 推論統計：假設檢定（t 檢定、卡方檢定）、信賴區間
- 資料視覺化：直方圖（分布）、散佈圖（相關性）、箱型圖（離群值）、折線圖（趨勢）

### 資料隱私與安全

| 面向 | 措施 |
|------|------|
| **資料隱私** | 去識別化、匿名化、差分隱私（Differential Privacy） |
| **存取控制** | RBAC 角色權限管理、多因素驗證（MFA）、最小權限原則 |
| **傳輸安全** | TLS/SSL 加密；靜態儲存加密 |
| **法規合規** | GDPR（歐盟）、CCPA（加州）、台灣個資法 |

## 3. 機器學習概念

### 三大學習類型

| 類型 | 訓練資料 | 目標 | 常見演算法 | 應用 |
|------|---------|------|----------|------|
| **監督式學習** | 有標籤 | 分類或迴歸 | 邏輯迴歸、決策樹、SVM、KNN、隨機森林 | 醫療診斷、信用評分 |
| **非監督式學習** | 無標籤 | 探索結構 | K-Means、DBSCAN、PCA | 顧客分群、異常偵測 |
| **強化學習** | 環境回饋 | 最大化累積獎勵 | Q-Learning、DQN、PPO | 遊戲 AI、自動駕駛 |

**機器學習流程：**
```
定義問題 → 資料收集與清洗 → 選擇與建立模型 → 訓練與評估 → 分析結果修正 → 最終模型輸出
```

### 模型評估指標

**分類模型：**

| 指標 | 說明 | 適用場景 |
|------|------|---------|
| **Accuracy** | 預測正確比例 | 類別平衡時 |
| **Precision** | 預測為正中實際為正的比例 | 降低誤報（如垃圾郵件）|
| **Recall** | 實際為正中被正確預測的比例 | 降低漏報（如疾病診斷）|
| **F1-Score** | Precision 與 Recall 的調和平均 | 不平衡資料 |
| **ROC/AUC** | 衡量分類器整體區分能力 | 模型效能整體評估 |

**迴歸模型：** MSE、RMSE、MAE、R²（決定係數）

### 過擬合與欠擬合

| 問題 | 成因 | 解法 |
|------|------|------|
| **過擬合（Overfitting）** | 模型太複雜，記住訓練資料雜訊 | 正規化（L1/L2）、Dropout、Early Stopping、增加資料 |
| **欠擬合（Underfitting）** | 模型太簡單，無法捕捉規律 | 增加模型複雜度、增加特徵、減少正規化 |

> **陷阱：降低學習率（Learning Rate）無法解決過擬合。**

## 4. 鑑別式 AI 與生成式 AI

### 比較對照

| | **鑑別式 AI（Discriminative AI）** | **生成式 AI（Generative AI）** |
|--|----------------------------------|------------------------------|
| **目標** | 判斷與分類，學習決策邊界 | 創造新內容，學習資料內在分布 |
| **輸出** | 類別、數值、標籤 | 文字、圖像、音訊、程式碼 |
| **技術** | 邏輯迴歸、SVM、決策樹、CNN（分類）| GAN、VAE、Diffusion、GPT |
| **應用** | 垃圾郵件過濾、信貸評分、詐欺偵測 | 文字生成（ChatGPT）、圖像生成（Midjourney）|

**協同應用：** 在醫療影像中，生成式 AI（GAN）可生成模擬資料，增強鑑別式 AI 的準確性與泛化能力。

### 深度學習核心技術

| 概念 | 說明 |
|------|------|
| **前向傳播** | 輸入資料逐層計算，產生預測輸出 |
| **反向傳播** | 計算損失函數，透過梯度下降更新權重 |
| **梯度下降** | 常見變體：SGD、Adam、RMSProp；需避免梯度消失/爆炸 |
| **激活函數** | 引入非線性：ReLU（主流）、Sigmoid、Tanh、GELU |

**主要深度學習架構：**
- **CNN**：影像分類、目標偵測
- **RNN/LSTM**：序列資料（語音、文字）
- **Transformer**：NLP 與多模態（GPT、BERT）
