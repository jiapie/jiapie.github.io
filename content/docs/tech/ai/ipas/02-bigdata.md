# 02.大數據處理分析與應用

## 1. 機率統計基礎

### 敘述性統計與資料摘要

敘述性統計（Descriptive Statistics）著重「將資料本身做最直接的總結與可視化」，目的是快速了解樣本的重要特徵與潛在模式，分三大面向：

**① 集中趨勢（Central Tendency）**

| 指標 | 定義 | 優點 | 限制 |
|------|------|------|------|
| **算術平均（Mean）** | 所有值的總和 ÷ 樣本數 | 計算直覺、普及度高 | 對極端值敏感 |
| **幾何平均（Geometric Mean）** | n 個數值的乘積開 n 次方 | 反映成長率/報酬率累積效果 | 不能有負值或零 |
| **調和平均（Harmonic Mean）** | n ÷ (各值倒數之和) | 速率/效率類數據更合理 | 不能有零或負值 |
| **中位數（Median）** | 排序後中間位置的值 | 不受極端值影響 | 無法呈現離散度 |
| **眾數（Mode）** | 出現頻率最高的值 | 適用於類別型資料 | 連續資料需先分桶 |

> 資料有極端值時，**中位數**比算術平均更能代表真實中心。

**② 離散程度（Variability）**

| 指標 | 公式 | 說明 |
|------|------|------|
| **變異數（Variance）** | σ² = Σ(xᵢ−μ)²/N | 衡量資料偏離平均的程度 |
| **標準差（SD）** | σ = √Variance | 與原資料同單位，直觀反映波動幅度 |
| **全距（Range）** | 最大值 − 最小值 | 簡單但受極端值影響 |
| **四分位距（IQR）** | Q3 − Q1 | 排除極端值，反映中間 50% 範圍 |

**箱形圖（Box Plot）解讀：**
- **箱體**：Q1（25%）→ Q3（75%），涵蓋中間 50% 資料
- **中線**：Q2（中位數）
- **鬚**：上鬚 = Q3 + 1.5×IQR；下鬚 = Q1 − 1.5×IQR
- **超出鬚的點**：離群值（Outlier）

**③ 分佈形狀（Shape）**

| 指標 | 說明 | 判斷標準 |
|------|------|---------|
| **偏度（Skewness）** | 分佈的對稱程度 | >0 右偏（長尾在右）；<0 左偏；≈0 對稱 |
| **峰度（Kurtosis）** | 尾部厚度與峰頂尖銳程度 | >3 高峰（重尾）；≈3 常態；<3 扁平 |

### 機率分佈與資料分佈模型

**機率分佈兩大類型：**

| 類型 | 描述工具 | 特性 |
|------|---------|------|
| **離散型** | 機率質量函數（PMF）| 值為有限或可數整數（如擲骰子、訂單數）|
| **連續型** | 機率密度函數（PDF）| 值為連續實數（如身高、體重、時間）|

**關鍵統計量：**
- **期望值 E(X)**：隨機變數的加權平均值（長期平均結果）
- **變異數 Var(X)**：衡量取值偏離期望值的程度；標準差為其平方根

---

**常見離散型分佈：**

| 分佈 | 描述情境 | 期望值 | 變異數 | 應用範例 |
|------|---------|-------|-------|---------|
| **伯努利分佈** | 單次二元試驗（成功/失敗）| p | p(1-p) | 廣告點擊（1/0）、品管合格（1/0）|
| **二項分佈** | n 次獨立試驗中成功次數 | np | np(1-p) | n 件產品中合格數、n 次廣告中點擊數 |
| **泊松分佈** | 固定時間/空間內事件發生次數（稀少事件）| λ | λ | 每小時客服來電數、設備故障次數 |

---

**常見連續型分佈：**

| 分佈 | 描述情境 | 期望值 | 變異數 | 應用範例 |
|------|---------|-------|-------|---------|
| **常態分佈 N(μ,σ²)** | 對稱鐘形曲線，自然界最普遍 | μ | σ² | 身高、體重、產品測量值；中央極限定理基礎 |
| **指數分佈** | 事件之間的等待時間（無記憶性）| 1/λ | 1/λ² | 機器故障間隔、客服等待時間 |
| **卡方分佈 χ²(k)** | 常態分佈平方和，偏態 | k | 2k | 假設檢定（適配度、獨立性）、變異數推論 |

> **中央極限定理（CLT）：** 當樣本量足夠大時，無論原始分佈為何，樣本平均數的分佈都趨近常態分佈——這是大數據統計推論的理論基石。

**分佈選擇指引：**
- 資料偏態（如收入、房價）→ 對數常態分佈
- 稀少且獨立事件 → 泊松分佈
- 等待時間 → 指數分佈
- 類別頻率比較 → 卡方分佈

### 假設檢定與統計推論

**統計推論兩大分支：**

| | **參數估計** | **假設檢定** |
|--|------------|------------|
| **目的** | 推測母體參數的值或區間 | 驗證某關於母體的主張是否成立 |
| **輸出** | 點估計 + 95% 信賴區間 | p 值 + 是否拒絕 H₀ |
| **範例** | 平均滿意度 4.2 分，95% CI = [4.0, 4.4] | 新藥 p=0.03 < 0.05 → 拒絕 H₀，新藥有效 |

**假設檢定五步驟：**

```
① 設定 H₀（虛無）與 H₁（對立）假設
② 選擇適當的檢定方法與統計量
③ 決定顯著水準 α（通常 0.05）
④ 計算檢定統計量與 p 值
⑤ 比較 p 值與 α，做出結論
```

**H₀ vs H₁：**
- **H₀（虛無假設）**：反映現狀、無差異的主張（如「新藥與安慰劑效果相同」）
- **H₁（對立假設）**：欲驗證的新主張（如「新藥效果不同」）
- 若 p ≤ α → 拒絕 H₀，支持 H₁；若 p > α → 無法拒絕 H₀

**顯著水準 α 選擇：**

| α | 容錯程度 | 適用場景 |
|---|---------|---------|
| **0.05** | 5% 誤判機率 | 最普遍使用的標準 |
| **0.01** | 1% 誤判機率 | 高風險決策（藥品療效、安全規範）|
| **0.10** | 10% 誤判機率 | 初步探索、假說生成階段 |

**常用假設檢定方法：**

| 情境 | 資料類型 | 方法 |
|------|---------|------|
| 單樣本平均是否與理論值不同 | 連續 | 單樣本 t 檢定 |
| 兩組獨立樣本平均是否不同 | 連續 | 獨立樣本 t 檢定 |
| 同一批人前後變化是否顯著 | 配對 | 配對樣本 t 檢定 |
| 三組以上平均是否不同 | 連續 | 單因子 ANOVA |
| 兩類別變數是否有關聯 | 類別（交叉表）| 卡方獨立性檢定 |
| 兩組比例是否有差異 | 比例 | 比例 Z 檢定 |
| 兩連續變數是否有線性關係 | 連續 | 皮爾森相關係數檢定 |

**兩種錯誤（必考）：**

| | H₀ 為真 | H₀ 為假 |
|--|---------|---------|
| **保留 H₀** | ✅ 正確（機率 1−α）| ❌ **型二錯誤**（漏判，機率 β）|
| **拒絕 H₀** | ❌ **型一錯誤**（誤判，機率 α）| ✅ 正確（檢定力 1−β）|

- **型一錯誤**：「寧可錯殺，不可放過」的代價（如詐欺系統誤判正常交易）
- **型二錯誤**：「寧可放過，不可錯殺」的代價（如診斷系統漏診真實疾病）
- **醫療診斷、安全風控** 等場景重視降低型二錯誤（提高 Recall）

## 2. 大數據處理技術

### 大數據與資料挖礦流程

**大數據 5V 特性：**

| 特性 | 說明 |
|------|------|
| **Volume（量）** | 資料量龐大（TB、PB 級） |
| **Velocity（速）** | 產生與處理速度極快（串流資料） |
| **Variety（多樣）** | 結構化、半結構化、非結構化資料 |
| **Veracity（真實）** | 資料品質與可信度 |
| **Value（價值）** | 從海量資料中提煉有用洞察 |

**KDD 資料挖礦流程（Knowledge Discovery in Databases）：**

```
資料選取 → 資料前處理 → 資料轉換 → 資料挖礦 → 結果評估與解釋
```

- **資料選取**：從資料庫中抽取與任務相關的子集
- **資料前處理**：清洗缺失值、雜訊、異常值
- **資料轉換**：正規化、聚合、特徵建構
- **資料挖礦**：套用演算法（分類、分群、關聯、迴歸）
- **結果評估**：評估模型效能，萃取可行知識

---

### Hadoop 與分散式處理平台

**Hadoop 核心架構：**

| 元件 | 功能 |
|------|------|
| **HDFS**（分散式檔案系統） | 將大檔案切割並分散儲存於多個節點，具備容錯複製機制 |
| **MapReduce** | 分散式計算框架：Map（分散處理）→ Reduce（彙整結果） |
| **YARN** | 資源管理排程器，協調叢集資源分配 |

**MapReduce 運作邏輯：**
- **Map 階段**：切分輸入資料，各節點平行處理 → 輸出 (key, value) 對
- **Shuffle & Sort**：按 key 整理並分派到 Reducer
- **Reduce 階段**：彙整相同 key 的值，輸出最終結果

**Hadoop 生態系重要工具：**

| 工具 | 用途 |
|------|------|
| **Hive** | SQL-like 查詢大數據（批次分析） |
| **HBase** | NoSQL 欄式資料庫，適合低延遲讀寫 |
| **Spark** | 記憶體內計算，比 MapReduce 快 10–100 倍，支援串流與 ML |
| **Kafka** | 高吞吐量分散式訊息佇列，適合即時串流資料 |

**為何重要：** 單一機器無法儲存或計算 PB 級資料；分散式平台解決了擴充性（Scale-out）與容錯問題，是現代大數據基礎建設的基石。

### 數據收集與清理

**資料前處理在資料分析中的關鍵角色：**

> 「垃圾進，垃圾出（Garbage In, Garbage Out）」—— 資料品質直接決定模型效能上限。

**前處理五大步驟：**

| 步驟 | 常見問題 | 解法 |
|------|---------|------|
| **缺失值處理** | 空白欄位影響統計與訓練 | 刪除、均值/眾數填補、KNN 插補 |
| **雜訊與異常值** | 量測誤差、輸入錯誤 | Z-score、IQR 法偵測；截斷或修正 |
| **特徵縮放** | 不同量綱影響距離計算 | 標準化（Z-score）、Min-Max 正規化 |
| **類別編碼** | 文字標籤無法直接計算 | One-Hot Encoding、Label Encoding |
| **資料不平衡** | 類別比例懸殊 | 過採樣（SMOTE）、欠採樣、調整類別權重 |

### 數據儲存與管理

**資料庫類型比較：**

| 類型 | 特性 | 代表技術 | 適用場景 |
|------|------|---------|---------|
| **關聯式（RDBMS）** | 結構化、ACID 事務保證 | MySQL、PostgreSQL | 金融交易、ERP 系統 |
| **NoSQL 文件型** | 靈活 Schema、JSON 格式 | MongoDB | 電商商品資料、日誌 |
| **NoSQL 欄式** | 高寫入吞吐量、分散式 | HBase、Cassandra | 物聯網時序資料 |
| **NoSQL KV 型** | 極高讀寫速度 | Redis | 快取、Session 管理 |
| **時序資料庫** | 針對時間序列優化 | InfluxDB | IoT 感測器、監控 |
| **資料倉儲（DW）** | 歷史資料 OLAP 分析 | Snowflake、BigQuery | 商業智慧報表 |
| **資料湖（Data Lake）** | 儲存所有格式原始資料 | AWS S3 + Glue | 機器學習資料集 |

**資料倉儲 vs 資料湖：**

| | **資料倉儲** | **資料湖** |
|--|------------|----------|
| **資料結構** | 結構化，已清洗 | 原始、各種格式 |
| **處理方式** | Schema-on-Write | Schema-on-Read |
| **使用者** | 商業分析師 | 資料科學家 |
| **查詢速度** | 快 | 較慢（需轉換）|

### 數據處理技術與工具

**批次 vs 串流處理：**

| | **批次處理（Batch）** | **串流處理（Streaming）** |
|--|-------------------|-----------------------|
| **特性** | 累積資料後一次處理 | 資料一到即時處理 |
| **延遲** | 高（分鐘/小時）| 低（毫秒/秒）|
| **工具** | Hadoop MapReduce、Spark | Kafka、Spark Streaming、Flink |
| **應用** | 每日報表、月結、ETL | 詐欺偵測、即時推薦、IoT 警報 |

**ETL vs ELT：**
- **ETL**（Extract→Transform→Load）：先在資料倉儲外部轉換，適合傳統資料倉儲
- **ELT**（Extract→Load→Transform）：先載入雲端資料湖，再按需轉換，適合現代雲端架構

**資料品質管理關鍵指標：**
- **完整性**：無缺失值
- **一致性**：跨系統資料定義統一
- **時效性**：資料的新鮮程度符合業務需求
- **準確性**：資料反映真實情況

## 3. 大數據分析方法與工具

### 統計學在大數據中的應用

**大數據中的特徵工程統計技術：**

| 技術 | 目的 | 方法 |
|------|------|------|
| **數值縮放（Scaling）** | 消除量綱差異，使特徵可比 | Min-Max 正規化（0–1）、Z-score 標準化（μ=0, σ=1）|
| **偏態修正** | 將偏態分佈轉為近似常態 | 對數轉換（log）、Box-Cox 轉換 |
| **離群值處理** | 降低極端值影響 | IQR 截斷法、Z-score 閾值法（|z|>3 視為離群值）|
| **分佈擬合** | 找最適合資料的機率模型 | 最大似然估計（MLE）、KS 檢定驗證擬合效果 |
| **相關性分析** | 識別冗餘特徵 | 皮爾森相關係數（線性）、Spearman 相關（非線性/序列）|
| **卡方特徵選擇** | 類別型特徵與目標的關聯 | 計算每個特徵的卡方統計量，選保留顯著特徵 |

**A/B 測試（大數據中的假設檢定應用）：**
- 常見於產品優化（介面改版、廣告素材、推薦演算法）
- 流程：設定 H₀（A、B 無差異）→ 隨機分組 → 收集指標 → 統計顯著性檢定（t 檢定/卡方）→ 決策
- 注意：樣本量需足夠大，避免過早停止（Peeking Problem）

### 關聯規則挖礦（Association Rule Mining）

**核心概念：** 從交易資料中找出「若 A 則 B」的共現規律。

**三大指標：**

| 指標 | 定義 | 說明 |
|------|------|------|
| **支持度（Support）** | P(A ∩ B) | 同時包含 A、B 的交易比例；反映規則的普遍性 |
| **信賴度（Confidence）** | P(B \| A) = P(A∩B)/P(A) | 買了 A 的人也買 B 的比例；反映規則的可靠性 |
| **提升度（Lift）** | Confidence / P(B) | >1 表示正相關；= 1 表示獨立；< 1 表示負相關 |

**Apriori 演算法：**
- 原理：若項目集不頻繁，其超集必然也不頻繁（向下封閉性）
- 流程：找出所有頻繁項目集 → 由頻繁項目集產生強關聯規則
- 設定最小支持度（min_support）與最小信賴度（min_confidence）門檻

**商業應用：**
- 購物籃分析（Beer & Diapers 經典案例）
- 交叉銷售、商品推薦、貨架陳列優化
- 網頁點擊流分析、醫療診斷共病規律

### 約略集合理論（Rough Set Theory, RST）

**核心概念：** 處理不精確、模糊、不完整資料的數學工具，無需額外假設資料的機率分佈。

**關鍵定義：**

| 概念 | 說明 |
|------|------|
| **上近似（Upper Approximation）** | 可能屬於目標概念的物件集合（寬鬆邊界） |
| **下近似（Lower Approximation）** | 確定屬於目標概念的物件集合（嚴格邊界） |
| **邊界域（Boundary Region）** | 上近似 − 下近似；即無法確定分類的模糊區域 |

**核心應用：**
- **特徵約簡（Attribute Reduction / Reduct）**：在不損失分類能力的前提下，找出最小必要特徵子集
- **規則生成**：從約簡後的資料中自動萃取 if-then 決策規則

**與模糊邏輯的差異：** 模糊邏輯處理隸屬度（程度問題），RST 處理知識的不精確性（邊界問題）。

**應用場景：** 醫療診斷（症狀特徵篩選）、信用評估、影像分類、資料前處理。

---

### 多變量分析與降維策略

**為何需要降維：**
- 高維資料存在「維度詛咒」：距離度量失效、計算量爆增、過擬合風險高
- 降維可去除冗餘特徵、加速訓練、便於視覺化

**主要降維方法：**

| 方法 | 類型 | 核心概念 | 適用場景 |
|------|------|---------|---------|
| **PCA**（主成分分析） | 線性、非監督 | 找最大方差方向，投影到正交主成分 | 一般降維、去雜訊 |
| **LDA**（線性判別分析） | 線性、監督 | 最大化類間距離、最小化類內距離 | 分類前的降維 |
| **t-SNE** | 非線性、非監督 | 保留局部鄰近結構，適合視覺化 | 2D/3D 視覺化高維資料 |
| **因素分析（FA）** | 線性、非監督 | 找隱含潛在因子解釋變數間的相關性 | 問卷資料、心理測量 |

**多變量分析常用方法：**
- **群集分析（Cluster Analysis）**：K-Means、階層式群集
- **判別分析（Discriminant Analysis）**：LDA 分類
- **典型相關分析（CCA）**：兩組變數間的相關結構

### 迴歸與時間序列預測

**迴歸分析：**

| 類型 | 說明 | 應用 |
|------|------|------|
| **簡單線性迴歸** | 一個自變數預測一個連續應變數 | 廣告費 → 銷售量 |
| **多元線性迴歸** | 多個自變數，檢查共線性（VIF） | 房價預測 |
| **多項式迴歸** | 非線性關係的線性化處理 | 曲線趨勢擬合 |
| **Logistic 迴歸** | 輸出機率（0–1），用於分類 | 信用評分、疾病診斷 |
| **Ridge / Lasso** | 加入正規化防止過擬合 | 高維特徵迴歸 |

**評估指標：** MAE、MSE、RMSE、R²（決定係數）

**時間序列分析：**

| 方法 | 特性 | 適用 |
|------|------|------|
| **移動平均（MA）** | 平滑短期波動 | 趨勢觀察 |
| **ARIMA** | 自迴歸整合移動平均；處理平穩/非平穩序列 | 股價、銷售量預測 |
| **指數平滑法（ETS）** | 近期資料賦予更高權重 | 需求預測 |
| **LSTM** | 深度學習捕捉長期依賴 | 複雜非線性時序 |

**時間序列四大成分：** 趨勢（Trend）、季節性（Seasonality）、循環（Cycle）、不規則（Irregular）

## 4. 大數據在 AI 的應用

### 大數據與機器學習

**大數據如何驅動機器學習：**

| 面向 | 說明 |
|------|------|
| **訓練資料規模** | 深度學習模型需要海量標注資料（如 ImageNet 1400 萬張圖片）才能達到高效能 |
| **特徵工程** | 大量原始資料提供更豐富的特徵選擇空間（使用者行為日誌、交易記錄等）|
| **模型更新** | 持續流入的新資料支援線上學習（Online Learning）和模型持續更新 |
| **資料增強** | 大數據提供多樣化樣本，降低過擬合風險 |

**大數據 + ML 的技術棧：**
```
資料收集（Kafka/API）
    ↓
資料儲存（HDFS/資料湖）
    ↓
資料處理（Spark ETL）
    ↓
特徵工程（Spark MLlib/Pandas）
    ↓
模型訓練（TensorFlow/PyTorch）
    ↓
模型部署（API/邊緣設備）
    ↓
監控回饋（MLflow/Grafana）
```


### 決策支持系統與最佳化技術

**決策支持系統（DSS, Decision Support System）：**
- 整合資料庫、模型庫與使用者介面，輔助管理者做出半結構化或非結構化決策
- 核心元件：資料管理子系統、模型管理子系統、使用者介面子系統

**DSS 類型：**

| 類型 | 說明 | 範例 |
|------|------|------|
| 資料導向型 | 以 OLAP、資料倉儲為基礎 | 銷售儀表板 |
| 模型導向型 | 以數學/統計模型為核心 | 財務模擬、供應鏈優化 |
| 知識導向型 | 結合規則引擎或 ML | 專家系統、推薦引擎 |

**最佳化技術：**

| 技術 | 說明 | 應用 |
|------|------|------|
| **線性規劃（LP）** | 線性目標函數 + 線性約束 | 生產排程、資源分配 |
| **整數規劃（IP）** | 決策變數為整數 | 人員排班、選址問題 |
| **遺傳演算法（GA）** | 模擬進化，適合組合最佳化 | 路徑規劃、特徵選擇 |
| **模擬退火（SA）** | 隨機搜尋，跳脫局部最優 | 排程、電路設計 |
| **粒子群最佳化（PSO）** | 群體智慧，協作搜尋 | 神經網路超參數調整 |

**在實務中的重要性：** DSS 將資料分析與最佳化模型整合，提供「what-if 情境分析」與最佳行動方案，廣泛應用於供應鏈管理、金融風控、醫療排程等領域。

---

### 大數據在鑑別式 AI 的應用

鑑別式 AI 依賴大數據進行監督學習，典型應用：

| 任務 | 大數據角色 | 代表場景 |
|------|---------|---------|
| **分類（Classification）** | 海量標注樣本訓練判別邊界 | 垃圾郵件過濾、信用審核、醫療診斷 |
| **預測（Regression）** | 歷史數據揭示趨勢模式 | 銷售預測、能耗預測、股價預測 |
| **異常偵測** | 正常行為模式的大量基準資料 | 詐欺交易偵測、網路入侵偵測 |
| **推薦系統** | 用戶互動行為日誌（點擊/購買）| 電商商品推薦、影音平台推薦 |
| **電腦視覺** | 大量標注影像訓練 CNN | 人臉辨識、工業瑕疵檢測 |

**關鍵挑戰：** 類別不平衡（如詐欺率 0.1%）→ 需 SMOTE 過採樣或調整類別權重；資料標注成本高 → 主動學習（Active Learning）優先選最有價值樣本標注。

### 大數據在生成式 AI 的應用

生成式 AI 的崛起高度依賴大規模語料與算力：

| 需求 | 說明 |
|------|------|
| **訓練語料規模** | GPT-3 使用 45TB 文本資料；LLaMA 2 使用 2T Tokens |
| **多模態資料** | 圖文配對（CLIP 用 4 億圖文對）、影音語料（多模態 LLM）|
| **資料多樣性** | 覆蓋多語言、多領域、多風格，提升模型泛化能力 |
| **資料清洗品質** | 低品質/有害內容過濾，是 LLM 訓練的關鍵成本 |

**RAG（檢索增強生成）與大數據結合：**
- 企業私有知識庫（結構化/非結構化文件）作為外部知識源
- 向量資料庫（Pinecone、Weaviate、pgvector）儲存嵌入向量
- 解決 LLM 知識截止日期問題，實現即時知識更新

**RLHF（人類回饋強化學習）的數據需求：**
- 需要大量人工標注的偏好資料（人類判斷哪個回答更好）
- ChatGPT 的對齊訓練即依賴此技術，資料標注品質直接影響模型安全性

### 大數據隱私保護、安全與合規

**隱私保護技術：**

| 技術 | 說明 | 適用場景 |
|------|------|---------|
| **去識別化（De-identification）** | 移除可直接識別個人的欄位（姓名、身分證）| 醫療資料釋出 |
| **匿名化（Anonymization）** | 資料與個人之間的關聯無法復原 | 開放資料集 |
| **假名化（Pseudonymization）** | 以代號取代真實識別碼，可逆但需金鑰 | 企業內部資料管理 |
| **差分隱私（Differential Privacy）** | 加入數學雜訊，使查詢結果無法還原個別記錄 | Google、Apple 統計資料收集 |
| **聯邦學習（Federated Learning）** | 資料不離開本地，只上傳模型梯度 | 醫療機構跨院訓練 |

**存取控制與安全：**
- **RBAC**（角色型存取控制）：依職位決定資料存取權限
- **最小權限原則**：只授予完成工作所需的最低存取權
- **加密**：傳輸中（TLS/SSL）+ 靜態儲存（AES-256）
- **稽核日誌**：記錄所有資料存取行為，支援事後追溯

**法規合規重點：**

| 法規 | 範圍 | 核心要求 |
|------|------|---------|
| **GDPR（歐盟）** | 歐盟居民資料 | 資料最小化、被遺忘權、明確同意 |
| **CCPA（加州）** | 加州居民資料 | 知情權、刪除權、拒絕銷售個資 |
| **台灣個資法** | 台灣個人資料 | 特定目的原則、當事人同意、安全維護措施 |
| **HIPAA（美國醫療）** | 醫療健康資料 | 傳輸中與靜態儲存均須技術防護措施 |

> **行政院生成式 AI 參考指引：** 禁止將敏感資料、個資、涉密文件直接輸入生成式 AI 系統；AI 產出僅能作為輔助，不可完全取代人類專業判斷。
